# Honesty Guardrails

## Core rule

Polish expression; do not fabricate substance. If a claim affects seniority, ownership, domain expertise, or business impact, require user evidence or mark it as an assumption to confirm.

## Allowed packaging

Allowed:
- Turn vague actions into PM language when grounded in user facts.
- Estimate ranges for small metrics only when the user provides a basis, e.g. `约`, `近`, `10%+`, `数十名用户`.
- Convert team results to candidate contribution wording, e.g. `参与推动`, `协助完成`, `支持落地`.
- Emphasize deliverables actually plausible for the role: PRD,原型,竞品分析,用户访谈纪要,需求池,数据看板,复盘报告.

Not allowed:
- Invent company, title, school, dates, awards, funding, patents, publications, or shipped products.
- Claim ownership of strategy, roadmap, P&L, architecture, algorithm design, or full project leadership without evidence.
- Add precise metrics such as `提升 37.6%` unless provided.
- Add regulated/high-risk domain claims, e.g. medical diagnosis, financial recommendation, legal compliance ownership, without explicit facts.

## Internship wording harness

For internship experiences, default to lower-ownership verbs:
- Preferred: `参与`, `协助`, `支持`, `负责其中的[模块]`, `配合`, `整理`, `输出`, `跟进`, `参与验收`.
- Use cautiously: `负责` only for a clear module, document, analysis, prototype, or small feature.
- Avoid unless proven: `主导`, `独立负责`, `从 0 到 1 搭建`, `制定产品战略`, `统筹多团队`, `最终决策`.

Replace over-claiming:
- Bad: `独立负责会员体系从 0 到 1 搭建。`
- Better: `参与会员体系 0-1 方案设计，负责权益页原型、竞品分析与需求文档撰写。`

## Metrics harness

Metric tiers:
1. Provided metric: keep it, optionally round for readability.
2. Derived metric: if raw before/after numbers are provided, calculate and label as approximate.
3. Plausible qualitative outcome: use when no metric exists, e.g. `提升需求评审效率`, `减少人工统计成本`, `支持版本按期上线`.
4. No outcome evidence: describe deliverable only; do not force a metric.

Use soft metric wording when uncertain:
- `约`, `近`, `超过`, `10%+`, `数十`, `百余`, `明显减少`, `显著改善` only if the direction is evidenced.

## Red flags to challenge

Ask a follow-up or soften wording when the user claims:
- A short internship “主导公司级核心项目”.
- No data access but large conversion metrics.
- No technical role but “设计推荐算法/训练模型”.
- No cross-functional context but “统筹研发、设计、运营、销售”.
- Very broad PM scope with no deliverables.
